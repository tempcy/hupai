由tempcy开发，在http://moni.51hupai.org/上测试。
请运行脚本：hupai.au3 / Please run script (x86)

                 ***基本原理***
================================================
识别当前屏幕上的价格和时间信息，根据预先设定的规则，进行半自动出价，验证码还是需要手动输入并确认。
================================================

                 ***准备工作***
================================================
1.安装AU3
www.autoitscript.com/site/autoit/downloads/
找到并点击Download AutoIt按钮。

2.建立字库
运行脚本后在相关画面内操作，须确认数字0-9、符号:和换行符_能被软件正确识别。

3.设定坐标
运行脚本后在相关画面内操作，须确认小键盘+、小键盘-、小键盘*功能正常。

================================================

                 ***拍牌流程***
================================================
『第二次出价』 ─> 【输入验证码】
               └> 【手动确认】
               └> 【取消出价框】

『第三次出价』 ─> 【输入验证码】（不要确认！）

『出价检查』┬《失败》─>『重新出价』
            │        └>【输入验证码】
            │        └>【手动确认】
            │
            └《通过》─>《满足设定时间或差额》
                      └>『自动确认』
================================================

                 ***可用热键***
================================================
『二次出价』          【F7】 （不用手动按）
『三次出价』          【F10】（不用手动按）
『价格阻塞时重新出价』【F8】 （不用手动按）
『价格超差时重新出价』【F9】 （不用手动按）
『刷新验证码』        【Num*】
『确认出价』          【Num+】
『取消出价/出价框』   【Num-】
================================================

                 ***参数说明***
================================================
    二次出价的目的是保底，反正有三次出价的机会，这里用一次可以应对价格完全跳不动的情况（大概1-2年才出现一次）。

    三次出价是应对跳价正常的出价，每个月价格曲线的最大影响因素是上个月跳价情况，以及各个拍牌网站发布的当月出价策略。强烈建议避开集中出价时间，价格可参考xls分析，如果上个月跳价卡死，这个月肯定要高估，然后逐月低估直到跳价再次卡死。

    补抢检查再次出价是需要拼速度的，手速慢建议设定在50秒，手速快可以在52秒，理论上越晚估价可信度越高。如果不想拼手速，请把检查条件放宽。

    跳价好比开车，油门掌握在每个竞标人的手里，只有那些正确命中了正出价区间的出价，才能推动跳价。但是，竞标人都想推最后那一把上车，把前面出价的人挤下车。前面推车人数不够，就会造成跳价卡死不动。总之，每个月跳价情况千差万别，没有一套万能的参数能保证你命中，脚本只能帮助你减少人为的操作失误。

    最后，祝各位好运！
